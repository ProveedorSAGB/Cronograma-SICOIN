📊 Actividad 1: Consolidación de Bases de Datos Históricas

Iniciado
Objetivo: Consolidar las bases de datos históricas del SICOIN (PTAR y PTCI vs PTAR_DESGLOSADO y PTCI_DESGLOSADO) mediante resectorización, cruce de riesgos y consolidación de acciones de control y mejora, conforme a las disposiciones de la Ley General del Sistema Nacional Anticorrupción.
Resectorización de Bases PTAR y PTCI
Reorganización sectorial de las bases principales (90% completado)
Resectorización PTAR_DESGLOSADO
Aplicación de criterios de resectorización a base desglosada
Cruce de Riesgos en PTAR
Identificación y mapeo de riesgos entre bases
Consolidación Acciones de Control
Unificación de acciones PTAR y PTAR_DESGLOSADO
Consolidación Acciones de Mejora PTCI
Armonización de acciones de mejora continua
Actualización SICOIN Dashboard
Migración de datos consolidados al sistema de visualización
Informe de Depuración
Documento técnico con registros a eliminar de la base original

🔍 Actividad 2: Análisis Integral y Propuestas de Mejora

Pendiente
Objetivo: Realizar un análisis exhaustivo del SICOIN para identificar variables críticas, proponer nuevos reportes, optimizar visualizaciones de datos y establecer criterios de normalización institucional conforme a las Disposiciones Generales de Control Interno.
Análisis de Variables Críticas
Identificación de variables que generan oportunidades de análisis
Propuesta de Nuevos Reportes
Diseño de reportes con variables adicionales no contempladas
Optimización de Visualizaciones
Mejora de cuadros de mando y dashboards existentes
Depuración de Módulos
Identificación y retiro de secciones obsoletas
Investigación Normativa
Requisitos legales para normalización del SICOIN
Propuesta de Normalización
Criterios obligatorios para carga institucional de datos

👥 Actividad 3: Gestión y Consolidación de Usuarios

Planificado
Objetivo: Consolidar y optimizar la gestión de claves de usuario del SICOIN, incluyendo la baja de claves inválidas y la implementación de controles de acceso mejorados, en cumplimiento con las disposiciones de seguridad informática institucional.
Auditoría de Claves Activas
Revisión integral de usuarios con acceso vigente
Identificación de Claves Inválidas
Detección de usuarios inactivos o con permisos obsoletos
Proceso de Baja de Usuarios
Eliminación controlada de accesos no válidos
Consolidación de Perfiles
Unificación y optimización de roles de usuario
Implementación de Controles
Nuevos mecanismos de seguridad y validación
